### 不适合数据开发日常使用的问题

#### 问题 1（高）：跨库/跨 Schema 血缘场景无 Golden Case

- **现象**：全部 13 个 Golden Case 使用 `default.default` 单 catalog/schema。但真实 SQL 中使用了至少 10+ 个不同的 database/schema：
  - `default`、`ihotel_default`、`pub`、`fuwu`、`pf_risk_control`、`temp`、`htemp`、`ihotel_default` 等
- **影响**：无法验证 `ihotel_default.dw_ihotel_abtest_index_searchlist_di` → `default.mdw_order_v3_international` 这种跨库字段追溯的正确性。跨库 JOIN 是多源数据血缘分析的核心场景，缺失此覆盖意味着**生产环境链路断裂**。
- **建议**：在 R03 补充至少 1 个跨库 Golden Case（如 TC14：orders@default JOIN users@ihotel_default），将公共元数据扩展到 2+ catalog。

#### 问题 2（中）：视图血缘场景缺失

- **现象**：无 CREATE VIEW 或 FROM view_name 的用例。
- **数据开发背景**：数仓中大量使用视图做数据权限隔离、指标口径统一。数据开发排查问题时，经常需要展开视图到基表的血缘链路。
- **建议**：在 R03~R04 补充 1 个视图血缘用例（如 `CREATE VIEW v_order_summary AS SELECT ... FROM orders JOIN users`）。

#### 问题 3（中）：Map/Array/Struct 复杂类型字段血缘未建模

- **现象**：真实 SQL 中大量使用 `ext_flag_map['active_promotion_flag']`（Map 取值）、`user_info['orig_device_id']`（嵌套 Map）、`LATERAL VIEW posexplode`（Array 展开）、`get_json_object(value, '$.ext.pos')`（JSON 路径）。但 Golden Case 中的 orders/users/products 元数据全部是标量类型（bigint/string/decimal/timestamp），**没有 Map/Array/Struct 类型字段**。
- **影响**：无法验证 `orders.ext_flag_map['promotion_deduction_amount']` 到 `pay_amount` 的血缘追踪。当前只能将 Map 字段整体标记为一个依赖节点，无法下钻到 key 级别。
- **建议**：在 P2 阶段的 TC 中增加 Map 类型字段的元数据定义，验证 key-level 血缘追溯。

#### 问题 4（中）：诊断码覆盖不足

- **现象**：目前仅覆盖 5 种 DiagnosticCode：UNKNOWN_TABLE、UNKNOWN_COLUMN、AMBIGUOUS_COLUMN、STAR_EXPANSION_USED、MISSING_PARTITION_FILTER。
- **缺失的诊断场景**（基于真实 SQL 推断）：
  - `UNKNOWN_DATABASE`：跨库场景下游库未注册
  - `TYPE_MISMATCH`：CAST 类型不兼容（如 `CAST('abc' AS BIGINT)`）
  - `JOIN_KEY_NOT_INDEXED`：JOIN 字段未在元数据中标记为 join_key，可能影响性能
  - `SELF_JOIN_RISK`：自连接未使用别名区分
  - `NON_DETERMINISTIC_UDF`：使用了非确定性 UDF（如 rand()）
  - `MISSING_PARTITION_FILTER`：已在 TC13 覆盖，但真实 SQL 中有更多变体
- **建议**：在 R03 补充诊断码清单，按优先级逐步覆盖。

#### 问题 5（低）：目录 `99_非业务元数据/` 中的 IDE 文件应归档

- **文件**：`测试用例/99_非业务元数据/.idea/` 下约 50 个 dataSources XML 和 meta 文件
- **问题**：README 已说明不参与测试，但 physical 文件占用目录空间。`.idea/dataSources/` 下的 75+ 个 `.meta` 文件增加了目录遍历开销。
- **建议**：要么移出测试用例目录（放入 `.gitignore` 忽略的本地配置目录），要么在 CI 脚本中显式排除。

#### 问题 6（低）：`测试用例/sql血缘/` 目录含义不清

- **文件**：`测试用例/sql血缘/前端页面` 和 `前端页面.tsx`
- **问题**：该目录包含前端 TSX 代码而非 SQL，与目录名称 "sql血缘" 不一致。`前端页面` 文件无扩展名，难以判断格式。
- **建议**：确认该目录用途——如果是前端血缘画布的参考截图/代码，应移到 `开发文档/` 而非 `测试用例/`。

#### 问题 8（低）：`AB实验_流量/拆分方案/` 与主目录存在文件重复

- **判断**：版本链保留有历史追溯价值（类似 Git history），但作为**业务测试用例**存在冗余——只需要 `数分终版sql.sql` 作为唯一测试输入。
