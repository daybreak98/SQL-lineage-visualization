# Agent 模块拆解总览与执行协议

> 来源：`..\00_主需求文档\sql_lineage_workbench_requirement_breakdown_v0.9.1.md`。
> 适用对象：编码 Agent / Code Review Agent / 测试 Agent。
> 交付原则：本模块只能在前置契约上做**增量实现**，不得重写已通过回归的上游模块。

## 1. 模块定位

| 字段 | 内容 |
|---|---|
| 模块编号 | `M00` |
| 阶段 | `ALL` |
| 前置依赖 | 无 |
| 交付粒度 | 适合 1 个 Agent 独立开发、测试、提交 PR |

## 2. 目标

建立所有模块的执行顺序、依赖关系、回归策略和 Agent 交付协议。该文档不是功能实现模块，而是整个模块包的入口。

## 3. 本模块做什么

- 给出模块执行顺序和依赖 DAG。
- 定义每个 Agent 开发前必须阅读的共同约束。
- 定义 PR 交付格式、测试报告格式和禁止破坏规则。
- 说明 P0/P1/P2/P3 的能力递进边界。

## 4. 本模块不做什么

- 不实现任何代码。
- 不替代具体模块文档。
- 不新增主需求文档以外的大模块。

## 5. 交付物

- 模块执行顺序表。
- Agent PR 模板。
- 全局回归命令建议。
- 跨模块禁止事项。

## 6. 对外契约 / 输入输出

无 API。此文档只作为开发协调协议。

## 7. 建议实现步骤

- 按 M01→M02→M03→M04→M06→M05→M07→M08→M09→M10→M11→M12→M13→M14→M15→M16→M17 完成 P0 最小闭环。
- 完成 P0 后再进入 M18→M22 的 P1 增强。
- 完成 P1 后再进入 M23→M24→M25→M26 的 P2 能力。
- P3 的 M27/M28 仅在前序稳定后启动。

## 8. 单元测试与集成测试

- 检查所有模块文档是否包含目标、依赖、测试和验收标准。
- 检查模块编号是否连续。

## 9. 回归测试要求

- 每新增或修改一个模块，都必须更新本索引。
- 任何 Agent 不能跳过前置模块修改下游代码。

## 10. 验收标准

- 索引中至少包含 10 个以上独立模块。
- 每个模块都有明确前置依赖和回归要求。

## 11. Agent 注意事项

Agent 接到任一模块时，应先读取 `00_AGENT_MODULE_INDEX.md`、本模块文档、以及所有前置模块的验收标准。

---

## 不可破坏的上游约束

- 不允许让 SQLGlot AST 直接暴露给前端。
- 不允许让前端自行推导或修正血缘。
- 不允许把 CTE、子查询、输出字段伪装成物理字段。
- 不允许把 unsupported 结构伪装成 success。
- 每次提交必须跑完本模块测试和所有前置模块回归。
