## 三、当前界面问题

### 3.1 Empty 态缺少可操作 CTA [P0]

**问题描述**：
- 当 SQL 编辑器为空时，TopBar 右侧仅显示灰色 muted 文本 "Paste SQL or Load Example"。
- 该文本不可点击，没有按钮样式，也没有与之关联的动作（如粘贴示例 SQL、加载 Demo）。
- 根据 §16.1：`empty → Paste SQL / Load Example → Empty / TopBar → 次主按钮`，empty 状态至少需要一个**可见的可点击按钮**来引导用户。

**影响**：
- 用户首次打开页面时，TopBar 右侧完全空白（仅有灰色提示文字），缺乏明确操作引导。
- 与 Canvas 区的 Empty Guide（有图标、标题、引导文案）相比，TopBar 区呈现"无操作可执行"的假象。

**截图要点**：
```
Header: [SQL Lineage | dialect ▼ | schema | separator | Format | Import |         Paste SQL or Load Example]
                                                                                      ^^^^^^^ 灰色文字，不可点击
```

### 3.2 btn-secondary 视觉区分度不足 [P1]

**问题描述**：
- `analyzed` 状态下 Analyze 按钮使用 `.btn-secondary` 类，该类的唯一样式差异是 `color: var(--color-text-muted)`。
- 按钮背景仍为 `var(--color-bg)`，边框仍为 `var(--color-border)`，与同行的 Format/Import 按钮**完全一样**。
- 根据 §16.2：`Analyze / Re-analyze 必须明显高于 Format、View、More`，即使 analyzed 后 Analyze 降权，也仍需保持可识别性。

**当前样式对比**：

| 按钮 | 背景 | 文字颜色 | 边框 | 权重感 |
|------|------|---------|------|--------|
| Analyze (ready, `.btn-primary`) | 主蓝 #2563eb | 白色 | 主蓝 | 高 |
| Analyze (analyzed, `.btn-secondary`) | 背景色 #fff | 灰色 #6b7280 | 灰色 #e5e7eb | **与 Format 完全相同** |
| Format (`.btn`) | 背景色 #fff | 正文色 #111827 | 灰色 #e5e7eb | 中 |
| Import (`.btn`) | 背景色 #fff | 正文色 #111827 | 灰色 #e5e7eb | 中 |

### 3.3 Editor analyzing 态无视觉反馈 [P1]

**问题描述**：
- 当 `analyzing` 时，`sql-textarea` 仅设置 `readOnly={true}`，没有任何视觉变化（背景、边框、叠加层）。
- 用户无法直观感知编辑器处于"正在分析中、不可编辑"的状态。
- 建议增加淡蓝色边框/背景或半透明遮罩层。

### 3.4 BottomPanel 折叠态缺乏内容提示 [P2]

**问题描述**：
- BottomPanel 折叠时仅显示一个 ▶ toggle arrow（32px 高）。
- 用户不知道面板内有 Diagnostics / Metadata / Raw Result 三个 Tab。
- 折叠态可显示 Tab 标签缩略（如 "Diagnostics · Metadata · Raw Result" 或计数 badge）。

### 3.5 BottomPanel 展开态 tabs 在 collapsed 时隐藏 [P2]

**问题描述**：
- 单击 toggle arrow 折叠 BottomPanel 时，Tab 按钮随 `bottomPanelOpen &&` 条件渲染而完全移除 DOM。
- 折叠态完全无视觉线索告知用户有 3 个 Tab 可用，降低了发现性。

## 四、用户注意力问题

### 4.1 Empty 态注意力引导缺失 [P0]

**问题**：Empty 态下，用户注意力被分散到三个区域：
1. TopBar 右侧灰色提示文字（不可操作）
2. Canvas 区 Empty Guide（📊 icon + 标题 + 引导文案）
3. SQL Editor 占位文本（"Paste your SQL here..."）

但三个区域之间缺乏从 Canvas → Editor 的明确视觉流向（如箭头、编号、动画引导）。§16.1 规定 empty 态应为"次主按钮"，而非分散的纯文本。

### 4.2 analyzed/dirty 态 Canvas 注意力引导合理 ✅

**通过**：Analyzed 后 TopBar CTA 降权（btn-secondary），Canvas 成为主要交互区（显示 "Lineage graph ready"），符合 §16.2 "analyzed 后 TopBar 不再是 L1；Canvas 变成 L1" 的设计意图。

### 4.3 dirty/failed 态注意力回到 TopBar ✅

**通过**：dirty 和 failed 态均正确将 Re-analyze 提升为 `btn-primary` + badge 提示，用户注意力被重新引导到 TopBar 主操作。

## 五、信息密度问题

### 5.1 TopBar 控件密度 ✅

**当前 TopBar 元素**：
```
[SQL Lineage] | [dialect ▼] [schema] | [Format] [Import]           [CTA]
```

控件数：品牌名 + 2 个输入 + 2 个辅助按钮 + 1 个 CTA 区域 = 6 个独立元素。

在 1366px 宽度下预算充足（约 1366 - 200(品牌) - 200(输入) - 200(辅助) - 200(CTA) = 566px 余量），信息密度合理。

### 5.2 Canvas 上方控件层数 ✅

**当前 Canvas 区**：
- 第 1 层：right-pane-label (28px) — "Lineage Graph" + 可能的 stale badge
- 第 2 层：canvas-placeholder 内容

M1 阶段 Canvas 上方仅 1 层 label，符合 §17.2 "Canvas 上方长期常驻控件最多两层" 的要求。后续 M5 加入 CanvasTopSearchBar 后需要严格控制不突破两层。

### 5.3 BottomPanel 展开态信息密度 ✅

3 个 Tab（Diagnostics / Metadata / Raw Result）+ 占位文案，密度合理。Tab 高度 24px、间距 2px、padding 0 10px，符合紧凑设计原则。

## 六、视觉层级问题

### 6.1 按钮权重层级 [P1]

当前按钮视觉层级：

| 层级 | 按钮 | 状态 | 可见度 |
|------|------|------|--------|
| L1 | Analyze/Re-analyze (btn-primary) | ready/dirty/failed | 高（蓝色实心） |
| L2 | Cancel (btn-danger) | analyzing | 高（红色边框 hover 变实心） |
| L3 | Format / Import (btn) | 所有态 | 中（灰色边框白色背景） |
| L3 | Analyze (btn-secondary) | analyzed | **低（与 Format 同级，且文字更淡）** |

**问题**：analyzed 态 Analyze 降级过度——它比同行 Format/Import 按钮更不显眼（文字 muted 灰色 vs 正文黑色），打破了"Analyze 在任意时刻应高于辅助按钮"的规则。

### 6.2 Stale/Error Badge 视觉层级 ✅

| Badge | 背景色 | 文字色 | 语义 |
|-------|--------|--------|------|
| Stale (⚠) | #fef3c7 (amber-50) | #92400e (amber-800) | 警告/过期 |
| Error (✕) | #fee2e2 (red-50) | #991b1b (red-800) | 错误/失败 |

两个 badge 使用了不同的颜色语义，视觉区分清晰。

### 6.3 Canvas 占位内容视觉层级 ✅

六个态的 Canvas 占位内容有明确区分：
- **empty**：📊 icon + 大标题 + 引导文案 → 引导型
- **ready**：标题 + 操作提示 → 等待型
- **analyzing**：标题 + loading 描述 → 进行中型
- **analyzed**：标题 + 说明文案 → 完成型
- **dirty**：stale badge + 过期提示 → 警告型
- **failed**：红色 ✕ + 红色标题 + 错误消息 → 错误型

### 6.4 SQL Editor 视觉层级 ✅

- 字体：等宽字体栈 `"Cascadia Code", "Fira Code", "JetBrains Mono", "Consolas", "Monaco"` 
- 字号：13px
- 背景：#fafbfc（略深于面板背景 #f9fafb）
- 无边框、无轮廓 → 编辑器区域视觉上清晰独立

### 6.5 节点视觉分类预留方向 [M4 前瞻性检查]

虽 M1 不要求渲染实际节点，但 CSS 架构应预留 `data-node-type` 和 `data-state` 属性支持。当前 CSS 中无节点相关 class，符合 M1 范围。

## 七、CSS 实现质量评估

### 7.1 CSS 变量体系 ✅

`index.css` 中定义的 CSS 变量覆盖了完整色板：
- 主色系：primary / danger / warning / success
- 背景系：bg / surface / border
- 文本系：text / text-muted
- 所有色系均提供 `-50` 浅色变体

**优点**：变量命名遵循语义化（`--color-primary` 而非 `--blue-500`），便于主题切换。

### 7.2 过渡动画 ✅

- `.btn`: `transition: background 0.15s, border-color 0.15s` — 快速响应
- `.splitter`: `transition: background 0.15s` — hover 即时反馈
- `.bottom-panel`: `transition: height 0.2s ease` — 展开/折叠平滑
- 动画时长均 ≤ 0.2s，不会造成感知延迟

### 7.3 可访问性 ⚠️

| 检查项 | 现状 | 建议 |
|--------|------|------|
| ARIA 标签 | 无 | 为 CTA 按钮、Splitter、BottomPanel toggle 添加 `aria-label` |
| 焦点指示器 | `.dialect-select:focus`, `.schema-input:focus` 有 `border-color` 变化 | 按钮缺少 `:focus-visible` 轮廓 |
| 键盘导航 | 无显式支持 | Splitter 不支持键盘拖拽（M7 可考虑） |
| 颜色对比度 | 未验证 | Stale badge (#92400e 在 #fef3c7 上) 符合 WCAG AA；Error badge 类似 |

### 7.4 性能 ✅

- 无运行时样式计算（纯 CSS class 切换）
- 无复杂 `box-shadow`、`filter`、`blur` 或循环动画
- `loading-dots` 使用 CSS `steps()` 而非 JS 计时器
- 符合 §12 中 "禁止节点使用复杂 box-shadow、filter、blur、渐变动画" 的前瞻要求
