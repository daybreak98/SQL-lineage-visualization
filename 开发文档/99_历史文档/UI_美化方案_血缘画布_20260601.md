## 二、用户注意力问题

| 问题 | 描述 |
|------|------|
| **注意力被同 Y 堆叠误导** | 当多个不同层级的节点挤在同一 Y 水平线时，用户无法快速判断数据流向（所有节点看起来像"同一行"），血缘图变成从左到右的一维线性排列，失去了层级递进的视觉节奏 |
| **物理表→CTE→输出 三层色彩跳跃不够** | 用户扫视图时，5 秒内应能区分"哪些是源头表、哪些是中间变换、哪些是最终输出"。当前 table(灰) vs CTE(浅蓝) vs output(主蓝) 的三层跳变虽有，但 CTE 的浅蓝与 output 的深蓝对比弱（strip 天蓝 vs 主蓝，差距仅色相微调） |

## 五、美化方案要点

### 5.1 Y 坐标自动分配（替代 `graph_builder.py` 当前逻辑）

**核心原则**：按**层级 (level)** 分组，同层多节点垂直均匀分布，层间 Y 不重叠。

**布局算法规格**：

| 参数 | 值 | 说明 |
|------|-----|------|
| `canvasPaddingTop` | 20px | 画布顶部内边距 |
| `canvasPaddingLeft` | 20px | 画布左侧内边距 |
| `levelGapX` | 200px | 层与层之间的 X 方向间距 |
| `nodeGapY` | 56px | 同层节点之间的 Y 方向间距（从上一个节点底部到下一个节点顶部） |
| `nodeHeight` | 36px (table) / 38px (cte) / 40px (output) | 节点实际高度 |
| `levelBaseX` | Level 0 起始 x=60 | 物理表位于最左 |

**层级分配规则**：

| 层级 | 节点类型 | X 坐标 | 说明 |
|------|---------|--------|------|
| Level 0 | `table` | `canvasPaddingLeft + 0 * levelGapX` = 20 | 物理表（数据源头），最左 |
| Level N (N≥1) | `cte`, `subquery` | `canvasPaddingLeft + N * levelGapX` | CTE/子查询按依赖深度排列 |
| Level Max | `output`, `output_field` | `canvasPaddingLeft + maxLevel * levelGapX` | 输出节点，最右 |

**Y 分配算法**：
1. 将同层节点按 `entityId` 稳定排序
2. 为该层计算 `maxY = previousLevelMaxY + nodeGapY`（确保层间不重叠）
3. 同层第一个节点 Y = maxY；后续节点 Y = 前一个节点底部 + `nodeGapY`
4. 最终画布高度 = 所有层中最大 Y 值 + `canvasPaddingBottom` + 50px 安全余量

**示例（mock 数据 6 节点）**：

```
Level 0 (table, x=20):   dwd_order_di  y=20
                         dim_user_df   y=20+36+56=112
Level 1 (cte, x=220):    order_base    y=20
Level 2 (subq, x=420):   valid_order_subq y=20
Level 3 (cte, x=620):    metric_base   y=20
Level 4 (output, x=820): Output Group  y=20
画布总宽：820+164+20=1004px；总高：112+36+20=168px
```

> **注意**：上例中 Level 1-4 各仅 1 节点，Y 均从 20 开始形成水平流。若某层有 3 个节点则垂直展开。

### 5.3 层级关系视觉 — 字号递进

**方案**：通过 `data-level` 属性，CSS 按层级设置 `font-size`。

| 层级 | 节点类型 | `font-size` | `font-weight` | 视觉效果 |
|------|---------|-------------|---------------|----------|
| Level 0 | table | **13px** | 750 | 最大——数据地基 |
| Level 1 | cte | **12px** | 700 | 中——一级变换 |
| Level 2 | subquery | **11.5px** | 650 | 次中——中间过滤 |
| Level 3+ | cte/subquery | **11px** | 600 | 小——深层嵌套 |
| Level Max | output | **13px** | 800 | 最大+最粗——终点锚点 |

**CSS 实现方向**（非代码，仅规格）：
```
.node[data-level="0"] .title  → font-size: 13px
.node[data-level="1"] .title  → font-size: 12px
.node[data-level="2"] .title  → font-size: 11.5px
.node[data-level="3"] .title,
.node[data-level="4"] .title,
...                           → font-size: 11px
.node[data-type="output"] .title → font-size: 13px; font-weight: 800
```

> **效果**：视觉上形成"从粗到细再到粗"的沙漏形层级感，引导视线从源头→中间→终点扫描。

### 5.4 边/箭头优化 — 虚实按端点类型区分

**当前状态**：
- `.edge.cte` → 实线 (stroke:#60A5FA) — 不区分两端节点类型
- `.edge.subq` → 虚线 (stroke:#A78BFA) — 同上

**改为按边两端节点类型区分**：

| 边类型 | 源节点 | 目标节点 | 线型 | 颜色 | 粗细 | 语义 |
|--------|--------|---------|------|------|------|------|
| table→cte | table | cte | **实线** | #94A3B8 (slate-400) | 1.4px | 物理表供给 CTE |
| table→table | table | table | **实线** | #CBD5E1 | 1.2px | 表间 JOIN |
| cte→cte | cte | cte | **虚线** (5,4) | #60A5FA (blue-400) | 1.7px | CTE 链式依赖 |
| cte→subq | cte | subquery | **虚线** (5,4) | #A78BFA (violet-400) | 1.7px | CTE 入子查询 |
| subq→cte | subquery | cte | **虚线** (5,4) | #60A5FA | 1.7px | 子查询出 CTE |
| *→output | 任意 | output | **实线** | #2563EB (blue-600) | 2px | 汇总到输出 |

**实现方向**：
- 后端 `LineageEdge` 需要带 `source_type` / `target_type` 信息（或前端从 `byEntity` 推导）。
- 前端 CSS 可使用组合选择器：基于边的 `data-source-type` / `data-target-type` 属性来控制线型。
- 当前 Marker 箭头：默认灰色 `#94A3B8`；当边为 `*→output` 或 `current` 时切换为主蓝 `#2563EB`。

### 5.5 间距与画布优化

| 参数 | 当前值 | **建议值** | 说明 |
|------|--------|-----------|------|
| 同层节点间距 (nodeGapY) | 无（全局 index * 58） | **56px** | 紧凑但不拥挤 |
| 层间距 (levelGapX) | 无（按 type 分 x） | **200px** | 保证边有足够曲率 |
| 画布 padding-top | 0 | **20px** | 避免顶部节点贴边 |
| 画布 padding-left | 0 | **20px** | 避免左侧节点贴边 |
| 画布 padding-bottom | 0 | **20px** | 底部留白 |
| 画布宽度 `.stage` | 1220px 固定 | **动态计算**：`maxLevel * levelGapX + maxNodeWidth + 2 * padding` | 适配不同规模血缘图 |
| 画布高度 `.stage` | 560px 固定 | **动态计算**：`maxY + paddingBottom + 50` | 适配多节点层 |
| 网格背景 | 18px 间距 | **20px 间距** | 与 padding/间距对齐 |
