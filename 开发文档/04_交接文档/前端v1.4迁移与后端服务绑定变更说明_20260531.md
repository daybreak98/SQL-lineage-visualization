## 二、本次主要变更

### 1. 前端工程切换

将 `sql-lineage-workbench-v1.4/src` 迁入 `frontend/src`，使 `frontend/` 成为 v1.4 React 页面主线。

同时更新：

```text
frontend/index.html
frontend/package.json
frontend/package-lock.json
frontend/tsconfig.json
frontend/tsconfig.node.json
frontend/vite.config.ts
frontend/postcss.config.js
frontend/tailwind.config.js
frontend/public/
```

### 2. 保留 Vite API 代理

`frontend/vite.config.ts` 已启用 React 插件，并保留：

```ts
server: {
  port: 5173,
  host: '0.0.0.0',
  proxy: {
    '/api': 'http://localhost:8000',
  },
}
```

这意味着前端代码统一使用相对路径 `/api/...`，不要硬编码 `http://localhost:8000`。

### 3. 新增前端 API client

新增：

```text
frontend/src/api/client.ts
```

封装了当前后端已提供的能力：

| 前端函数 | 后端接口 | 用途 |
|---|---|---|
| `getHealth()` | `GET /api/health` | 探测后端状态 |
| `formatSql(sql, dialect)` | `POST /api/sql/format` | SQL 格式化 |
| `analyzeSql(sql, dialect)` | `POST /api/sql/analyze` | SQL 解析 / 当前分析 stub |
| `previewMetadata(payload)` | `POST /api/metadata/import/preview` | 元数据导入预览 |
| `commitMetadata(payload)` | `POST /api/metadata/import/commit` | 元数据导入提交 |
| `listMetadataTables()` | `GET /api/metadata/tables` | 查询已导入表 |
| `listMetadataColumns(table)` | `GET /api/metadata/columns` | 查询字段 |

后续新增 API 时应继续扩展 `frontend/src/api/client.ts`，不要在组件里散落 `fetch`。

### 4. Format 按钮已绑定后端

位置：

```text
frontend/src/App.tsx
```

`TopBar` 中的 `Format` 按钮现在调用：

```ts
formatSql(sql, dialect)
```

并用后端返回的 `formatted_sql` 更新编辑器内容。

### 5. Analyze 按钮已绑定后端

位置：

```text
frontend/src/App.tsx
```

`Analyze` 按钮现在调用：

```ts
analyzeSql(sql, dialect)
```

当前后端 `/api/sql/analyze` 仍是 R04a 前的 stub，返回：

```text
tables_extracted
columns_extracted
diagnostics_report
summary
```

前端已将这些字段转换为：

```text
backendGraph
backendSearchItems
backendDiagnostics
```

并接入现有画布、搜索栏、诊断 Drawer。

### 6. Metadata 弹窗已接后端

新增：

```text
frontend/src/components/MetadataDialog.tsx
```

入口：

```text
TopBar -> Metadata 按钮
```

能力：

- 粘贴 JSON payload
- 调用 preview
- 调用 commit
- 展示后端 diagnostics
- 展示已导入 tables
- 点击表后展示 columns

注意：后端 `changes` 的表名/字段名来自：

```text
change.object_ref.table
change.object_ref.column
```

不要再读取旧单文件 HTML 里的 `change.table` / `change.column`。

### 7. 顶部 metadata badge 已接后端

`TopBar` 的 metadata badge 现在来自：

```ts
getHealth()
listMetadataTables()
```

示例显示：

```text
backend 0.1.0 · metadata: 3 tables
```

后端不可用时显示：

```text
metadata: backend offline
```

## 五、当前仍需注意的限制

### 1. `/api/sql/analyze` 仍是后端 stub

当前后端尚未完成 R04a 正式 `AnalysisResult + GraphViewModel`。

前端已预留优先消费：

```text
result.graph_view_model.nodes
result.graph_view_model.edges
```

如果后端正式返回 GraphViewModel，前端会优先使用它；否则继续使用当前 stub 的：

```text
tables_extracted
columns_extracted
```

生成临时图。

### 2. 旧 mock 数据没有完全删除

`frontend/src/data/mockLineage.ts` 仍保留，作用是：

- 页面默认展示
- 后端离线时保留视觉上下文
- 暂未由后端覆盖的详情、路径、示例 diagnostics

后续不要继续扩大 mock 逻辑；新增真实能力应优先走 API。

### 3. Metadata 弹窗目前是基础可用版

已支持 preview / commit / tables / columns，但还没有：

- 文件上传 JSON
- 大 payload 分页
- import_jobs 历史列表
- 后端错误码精细 UX 映射

这些应作为后续 UI/UX hardening 或 R02+ 增强处理。

### 4. 测试体系需要补前端测试

迁移后旧 `vitest` 相关配置被 v1.4 工程替换，目前只有 `build/typecheck`。

后续应补：

- API client 单元测试
- MetadataDialog 组件测试
- Format / Analyze 交互测试
- 后端离线 fallback 测试
