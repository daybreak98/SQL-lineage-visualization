## 三、Analyze API 当前能力

支持 P0 最小闭环：

- SQL parse
- FROM / JOIN 表与别名识别
- 元数据表解析
- qualified / unqualified 字段解析
- unknown table 诊断
- unknown column 诊断
- ambiguous column 诊断
- SELECT 直接字段投影
- SELECT alias
- SELECT literal
- function wrapper 的直接依赖字段
- 基础 LineageIR
- 后端 GraphViewModel
- stage_statuses
- source_locations
- diagnostics_report

暂不支持：

- CTE / 子查询完整血缘
- select * 展开
- CASE / window / aggregate 完整语义
- SemanticsReport
- 多视图 GraphViewModel
- SQL 与画布双向精确定位

这些仍应按模块路线进入 M18+ / P1 / P2。

## 六、后续开发注意事项

1. 后续不要恢复旧 `/api/sql/analyze` stub。
2. 前端不要自行推断最终字段血缘。
3. 后续新增复杂 SQL 能力时，应按 M18/M19/M20 等模块推进。
4. GraphViewModel 不应混入前端交互状态，如 selected、viewport、collapsed。
5. 修改 `AnalysisResult` 字段前必须同步后端测试和前端 TS 类型。
6. P0 Golden Case 仍需进入 M17 统一回归。
