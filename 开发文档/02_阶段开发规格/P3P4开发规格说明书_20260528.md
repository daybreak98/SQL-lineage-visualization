# SQL 血缘解析工作台｜P3 / P4 开发规格说明书

生成日期：2026-05-28

## 1. 文档定位

本文档覆盖 P3 / P4 阶段需求，可直接交给 agent 在 P0-P2 工程基础上实现。

P3 范围：

```text
R15 SQL 当前版本 diff 与重写辅助
R16 分析历史与快照
```

P4 范围：

```text
R18 平台化扩展预留
```

P3 要求可实现；P4 以扩展边界、接口占位和最小可用骨架为主，不要求完整企业级对接。

## 2. P3 成功标准

```text
1. 能对比当前页面两段 SQL 的 AnalysisResult 差异。
2. 能输出字段、血缘边、口径、诊断、图谱的变更摘要。
3. 能生成可解释的 SQL 重写建议。
4. 能保存分析历史与 AnalysisResult 快照。
5. 能读取历史快照并进行历史版本 diff。
6. 前端提供 DiffPanel、RewritePanel、AnalysisHistoryPanel。
```

```python
class DiffObjectType(str, Enum):
```

### 3.3 重写建议模型

```python
class RewriteSuggestionType(str, Enum):
    expand_star = "expand_star"
    qualify_column = "qualify_column"
    normalize_alias = "normalize_alias"
    remove_unused_column = "remove_unused_column"
    simplify_cte = "simplify_cte"
    add_partition_filter = "add_partition_filter"
    explain_risk = "explain_risk"


class RewriteSafety(str, Enum):
    safe = "safe"
    needs_review = "needs_review"
    unsafe = "unsafe"


class RewriteSuggestion(StrictBaseModel):
    suggestion_id: str
    suggestion_type: RewriteSuggestionType
    title: str
    description: str
    safety: RewriteSafety
    before_sql: str | None = None
    after_sql: str | None = None
    source_location_id: str | None = None
    related_entity_ids: list[str] = Field(default_factory=list)
    diagnostics: list[Diagnostic] = Field(default_factory=list)


class RewriteSuggestionRequest(StrictBaseModel):
    sql: str = Field(min_length=1)
    dialect: Dialect = Dialect.spark
    default_catalog: str = "default"
    default_schema: str = "default"
    metadata_version: str = "latest"
    case_sensitive: bool = False


class RewriteSuggestionResponse(StrictBaseModel):
    status: AnalysisStatus
    suggestions: list[RewriteSuggestion] = Field(default_factory=list)
    diagnostics: list[Diagnostic] = Field(default_factory=list)
```

安全规则：

```text
1. expand_star：safe，但必须用户确认后应用。
2. qualify_column：safe，如果字段唯一可解析。
3. normalize_alias：needs_review。
4. remove_unused_column：needs_review。
5. simplify_cte：needs_review。
6. add_partition_filter：unsafe，只给建议，不自动改 SQL。
```

## 4. 后端服务规格

### 4.1 AnalysisDiffService

新文件：

```text
backend/app/services/analysis_diff_service.py
```

公开接口：

```python
class AnalysisDiffService:
    def diff_results(
        self,
        left: AnalysisResult,
        right: AnalysisResult,
    ) -> AnalysisDiffResult:
        ...

    def diff_current_sql(
        self,
        request: CurrentSqlDiffRequest,
        analyzer: AnalysisOrchestrator,
    ) -> AnalysisDiffResult:
        ...

    def diff_snapshots(
        self,
        request: SnapshotDiffRequest,
        repository: AnalysisRepository,
    ) -> AnalysisDiffResult:
        ...
```

Diff 规则：

| 对象 | key |
|---|---|
| 输出字段 | output node id |
| 源字段 | source column node id |
| 血缘边 | edge id |
| metric | metric name |
| filter | normalized expression_text |
| join | join_id 或 join condition text |
| diagnostics | code + source_location_id + message |

严重程度：

| 变化 | severity |
|---|---|
| 删除输出字段 | breaking |
| 输出字段来源变化 | breaking |
| 指标公式变化 | breaking |
| 过滤条件变化 | warning |
| 新增诊断 error | breaking |
| 新增 warning | warning |
| 新增字段 | info |

### 4.2 RewriteService

新文件或增强：

```text
backend/app/services/rewrite_service.py
```

公开接口：

```python
class RewriteService:
    def suggest(self, request: RewriteSuggestionRequest) -> RewriteSuggestionResponse:
        ...

    def suggest_from_analysis(self, result: AnalysisResult) -> RewriteSuggestionResponse:
        ...
```

建议生成规则：

| 条件 | suggestion_type |
|---|---|
| SQL 包含 `*` 且可展开 | expand_star |
| UNKNOWN_COLUMN 由字段歧义导致 | qualify_column |
| 输出别名缺失或不规范 | normalize_alias |
| CTE 输出字段未被下游使用 | remove_unused_column / simplify_cte |
| 缺少分区过滤风险 | add_partition_filter |
| UDF 黑盒 | explain_risk |

### 4.3 AnalysisRepository

新文件：

```text
backend/app/repositories/analysis_repository.py
```

公开接口：

```python
class AnalysisRepository:
    def save(self, request: SaveAnalysisRequest) -> SaveAnalysisResponse: ...
    def get_snapshot(self, analysis_id: str) -> AnalysisSnapshot | None: ...
    def list_history(
        self,
        keyword: str | None = None,
        limit: int = 50,
        offset: int = 0,
    ) -> AnalysisHistoryListResponse: ...

```

存储要求：

```text
1. AnalysisResult 以 JSON 存储。
2. 保存 sql_hash、metadata_version、status、created_at。
3. 读取快照时必须返回当时的 AnalysisResult，不重新分析。
4. 删除历史记录必须同时删除 snapshot。
```

## 5. API 规格

### 5.1 `POST /api/sql/diff`

用途：对比当前页面两段 SQL。

请求：`CurrentSqlDiffRequest`

示例：

```json
{
  "left": {
    "sql": "select order_no from default.order_table",
    "dialect": "spark",
    "default_catalog": "default",
    "default_schema": "default",
    "metadata_version": "latest",
    "case_sensitive": false
  },
  "right": {
    "sql": "select order_no, user_id from default.order_table",
    "dialect": "spark",
    "default_catalog": "default",
    "default_schema": "default",
    "metadata_version": "latest",
    "case_sensitive": false

    "include_graph": true,
    "include_semantics": true,

    "include_source_location": true,

}
```

## 7. 前端实现规格

### 7.1 新增组件


### 7.2 DiffPanel

功能：

```text
1. 支持“当前 SQL vs 修改前 SQL”。
2. 展示 summary：新增、删除、变化、breaking。
3. 按 object_type 分组展示 changes。
4. 点击 change 高亮对应图节点或 SQL 位置。
5. breaking 变化使用 error 样式。
```

状态：

```ts
export interface DiffState {
  left_sql: string;
  right_sql: string;
  request_state: RequestState;
  current_diff: AnalysisDiffResult | null;
  selected_change_id: string | null;
}
```

### 7.3 RewritePanel

功能：

```text
1. 调用 /api/sql/rewrite/suggestions。
2. 按 safety 分组展示建议。
3. safe 建议可一键应用；needs_review 必须显示 diff 后让用户确认；unsafe 只显示说明，不提供自动应用。
4. 应用建议前保存当前 SQL 到 diff left_sql。
```

状态：

```ts
export interface RewriteState {
  request_state: RequestState;
  suggestions: RewriteSuggestion[];
  selected_suggestion_id: string | null;
  preview_after_sql: string | null;
}
```

### 7.4 AnalysisHistoryPanel

功能：

```text
1. 展示历史列表。
2. 支持按 keyword 搜索。
3. 点击历史项加载快照。
4. 支持删除历史。
5. 支持选择两个历史项进行 diff。
```

状态：

```ts
export interface HistoryState {
  request_state: RequestState;
  items: AnalysisHistoryItem[];
  selected_analysis_id: string | null;
  compare_left_id: string | null;
  compare_right_id: string | null;
}
```

### 7.5 保存策略

P3 前端提供两种保存：

```text
1. 手动保存：用户点击“保存分析”。
2. 自动保存可选：分析成功后提示，不默认保存，避免历史膨胀。
```
