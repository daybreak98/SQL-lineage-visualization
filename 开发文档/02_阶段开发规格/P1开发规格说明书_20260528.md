生成日期：2026-05-28

## 1. 文档定位

本文档覆盖 P1 阶段剩余需求，可直接交给 agent 在现有 P0 工程基础上实现。

P1 范围：

```text
R09b SourceLocation 精准提取
R10 join / CTE / 子查询基础支持
R11a select * 展开
R11b SQL completion
R11c SQL hover
R09c SQL 与图谱双向联动
```

P1 目标不是做完整语义分析，而是让真实数仓 SQL 的基础结构可解析、可定位、可交互。

### 2.2 P1 成功标准

```text
1. CTE 输出字段可以回溯到源表字段。
2. from 子查询输出字段可以回溯到内部源字段。
3. join 条件可以生成 join_condition 影响边。
4. select * / alias.* 可以基于元数据展开。
5. SQL 字段、表、表达式有可用 SourceLocation。
6. Monaco completion 可提示表名、字段名、常见函数。
7. Monaco hover 可展示字段/表元数据。
8. 点击图节点可定位 SQL，选中 SQL 可高亮图节点。
```

### 3.3 SourceLocation 精度要求

P1 的 `SourceLocation` 不改字段，但实现必须满足：

| 对象 | 位置要求 |
|---|---|
| select 字段 | 至少 line/col 和 raw_text，confidence >= 0.8 |
| 输出别名 | 能区分源字段和 alias，confidence >= 0.7 |
| from 表 | line/col 和 raw_text，confidence >= 0.8 |
| join 条件 | 条件整体 raw_text，confidence >= 0.7 |
| CTE 名 | CTE 名位置，confidence >= 0.7 |
| 子查询 alias | alias 位置，confidence >= 0.7 |

无法定位时：

```text
1. 不抛异常。
2. source_location_id = null。
3. 生成 SOURCE_LOCATION_UNAVAILABLE info 诊断，或在 details 中记录低置信度原因。
```

```python
class SourceLocationService:
    def build_index(self, sql: str) -> SqlLocationIndex: ...
    def locate_raw_text(
        self,
        sql: str,
        raw_text: str,
        entity_id: str,
        entity_type: EntityType,
        occurrence: int = 1,
    ) -> SourceLocation | None: ...
```

实现要求：

```text
1. P1 可用 token / raw text scan，不强制依赖 SQLGlot AST 自带坐标。
2. offset 必须基于 Python 字符串 index。
3. line/col 从 offset 计算，line 和 col 都从 1 开始。
4. 同一字段多次出现时使用 occurrence 区分。
5. raw_text 匹配大小写按 request.case_sensitive 处理。
```

```python
class ScopeResolver:

        self,
        ast: Any,
        request: AnalyzeSqlRequest,
        location_service: SourceLocationService,
    ) -> tuple[list[ScopeItem], list[SourceLocation], list[Diagnostic]]:

```

P1 必须支持：

```text
1. root select scope。
2. with cte as (...)，每个 CTE 生成 scope:cte:{name}。
3. from (...) alias，生成 scope:subquery:{alias}。
4. from table alias，建立 alias → table FQN。
5. join table alias，建立 alias → table FQN。
6. join condition，生成 JoinReference。
7. union all 可先只识别分支并生成 warning，不要求完整血缘。
```

字段解析优先级：

```text
1. 显式 alias.column：优先按 alias 定位来源。

3. 子查询 alias 输出字段：映射到子查询内部输出字段。
4. 无前缀字段：在当前 scope 可见字段中唯一匹配才成功。
5. 多个候选：AMBIGUOUS_COLUMN。
6. 无候选：UNKNOWN_COLUMN。
```


```python
class StarExpansionService:
    def expand_for_analysis(
        self,
        ast: Any,
        scopes: list[ScopeItem],
        metadata_context: MetadataContext,
    ) -> tuple[list[LineageNode], list[LineageEdge], list[Diagnostic]]:
        ...

    def rewrite_sql(
        self,
        request: SelectStarRewriteRequest,
    ) -> SelectStarRewriteResponse:
        ...
```

规则：

```text
1. select * from single_table：展开当前可见表全部字段，按 ordinal 排序。
2. select alias.*：只展开 alias 对应表字段。
3. 多表 select *：按 from/join 出现顺序拼接字段；重复输出名 P1 保留原名并生成 warning。
4. 元数据缺失：STAR_EXPANSION_FAILED error。
5. rewrite 不自动写回编辑器，只返回 rewritten_sql。
```

### 4.6 EditorIntelligenceService

新文件：

```text
backend/app/services/editor_intelligence_service.py
```

公开接口：

```python
class EditorIntelligenceService:
    def complete(self, request: EditorCursorContext) -> CompletionResponse: ...
    def hover(self, request: EditorCursorContext) -> HoverResponse: ...
```

completion 策略：

| 场景 | 返回 |
|---|---|
| `from ` 后 | 表名 |
| `join ` 后 | 表名 |
| `alias.` 后 | alias 可见字段 |
| select 列表中 | 当前 scope 可见字段 + 常见函数 |
| 空 SQL 或无法判断 | 常见关键字 + 函数 |

hover 策略：

| 场景 | 返回 |
|---|---|
| 光标在表名上 | 表注释、字段数量 |
| 光标在字段上 | 字段类型、注释、来源表 |
| 光标在 alias.field 上 | alias 对应表字段 |
| 无法解析 | `item = null` + HOVER_TARGET_UNRESOLVED info |

### 7.3 图谱联动规则

| 交互 | 规则 |
