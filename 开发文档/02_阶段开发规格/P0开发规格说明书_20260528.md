# SQL 血缘解析工作台｜P0 开发规格说明书

生成日期：2026-05-28

## 1. 文档定位

本文档把 P0 从“需求描述”落到“可开发规格”。开发人员或 agent 可以据此创建后端 Pydantic 模型、前端 TypeScript 类型、P0 API、Golden Case 测试断言和前端交互状态。

本文档只覆盖 P0，不展开 DDL。数据库表结构另开文档处理。

P0 主线：

```text
JSON 元数据导入
  → SQLite 元数据可查询
  → Monaco SQL 输入 / 格式化 / 提交分析
  → SQLGlot 解析
  → ScopeResolver
  → MetadataService
  → NameResolver
  → LineageIR
  → DiagnosticsReport
  → GraphViewModel
  → React Flow 基础画布
```

P0 明确不做：

```text
复杂 CTE
深层子查询
完整 select * 展开
completion
hover
完整表达式级血缘
完整查询口径分析
多视图切换实现
SQL diff
历史快照
真实 SQL 执行
```

P0 可以预留但不要求完整实现：

```text
SourceLocation 数据模型
GraphViewModel 多视图字段
SemanticsReport 空壳
expression 节点类型
diagnostics 诊断视图类型
```

## 2. 通用契约

### 2.1 命名约定

| 项 | 约定 |
|---|---|
| API JSON 字段 | 统一使用 `snake_case` |
| Python 字段 | 与 API JSON 一致，使用 `snake_case` |
| TypeScript 字段 | 与 API JSON 一致，使用 `snake_case` |
| 时间 | ISO 8601 字符串，例如 `2026-05-28T10:00:00+08:00` |
| ID | 业务可读、稳定、可测试，不使用随机 ID 作为 Golden Case 断言主体 |
| 空数组 | 使用 `[]`，不使用 `null` |
| 可缺省对象 | P0 可用 `null`，但字段必须保留 |
| 错误码 | 只使用枚举值，不用自由字符串 |

### 2.2 P0 稳定 ID 规则

Golden Case 和前端图谱依赖稳定 ID。P0 统一按以下规则生成：

| 对象 | ID 规则 | 示例 |
|---|---|---|
| root scope | `scope:root` | `scope:root` |
| 表节点 | `table:{catalog}.{schema}.{table}` | `table:default.default.order_table` |
| 表别名 | `alias:{scope_id}:{alias}` | `alias:scope:root:a` |
| 源字段节点 | `column:{catalog}.{schema}.{table}.{column}` | `column:default.default.order_table.order_no` |
| 输出字段节点 | `output:{scope_id}.{column}` | `output:scope:root.order_no` |
| 表达式节点 | `expr:{scope_id}.{output_column}` | `expr:scope:root.commission` |
| unknown 字段节点 | `unknown:{scope_id}.{column}` | `unknown:scope:root.missing_col` |
| 诊断 | `diag:{code}:{index}` | `diag:UNKNOWN_COLUMN:1` |
| 位置 | `loc:{entity_id}` | `loc:output:scope:root.order_no` |
| 边 | `edge:{edge_type}:{source}->{target}` | `edge:projection:column:default.default.order_table.order_no->output:scope:root.order_no` |

### 2.3 状态语义

| 状态 | 使用场景 |
|---|---|
| `success` | SQL 解析、元数据解析、字段血缘均成功 |
| `partial` | 解析链路部分成功，但存在 unknown、ambiguous 或 P0 不支持能力 |
| `failed` | 语法解析失败、元数据完全不可用、请求无法形成可靠结果 |

规则：

```text
1. SQL 语法错误：HTTP 200 + AnalysisResult.status = failed + PARSE_ERROR。
2. 请求体结构错误：HTTP 422，由 FastAPI / Pydantic 返回。
3. 未知表：如果所有输入表都未知，status = failed；如果只有部分未知，status = partial。
4. 未知字段：status = partial。
5. 字段歧义：status = partial。
6. P0 未支持的语义分析：不算失败，SemanticsReport.status = not_supported_in_p0。
```

```python
from __future__ import annotations

from typing import Any, Literal

from pydantic import BaseModel, ConfigDict, Field, field_validator

    partial = "partial"

    scope = "scope"

    column = "column"

    expression = "expression"
    filter = "filter"
    join = "join"
    cte = "cte"
    subquery = "subquery"

    statement = "statement"
    scope = "scope"
    table = "table"
    column = "column"

    table = "table"
    column = "column"
    expression = "expression"
    semantics = "semantics"
    diagnostics = "diagnostics"


class GraphNodeType(str, Enum):
    table = "table"
    column = "column"
    output_column = "output_column"
    expression = "expression"
    unknown = "unknown"
    diagnostic = "diagnostic"


class GraphEdgeType(str, Enum):
    projection = "projection"
    alias = "alias"
    expression = "expression"
    filter_condition = "filter_condition"
    unknown = "unknown"


class DiagnosticLevel(str, Enum):
    error = "error"
    warning = "warning"
    info = "info"


class DiagnosticCode(str, Enum):
    PARSE_ERROR = "PARSE_ERROR"
    UNKNOWN_TABLE = "UNKNOWN_TABLE"
    UNKNOWN_COLUMN = "UNKNOWN_COLUMN"
    AMBIGUOUS_COLUMN = "AMBIGUOUS_COLUMN"
    STAR_EXPANSION_FAILED = "STAR_EXPANSION_FAILED"
    UNSUPPORTED_DIALECT_FEATURE = "UNSUPPORTED_DIALECT_FEATURE"

    DUPLICATE_ALIAS = "DUPLICATE_ALIAS"
    METADATA_VERSION_MISMATCH = "METADATA_VERSION_MISMATCH"
    METADATA_IMPORT_INVALID_JSON = "METADATA_IMPORT_INVALID_JSON"
    METADATA_IMPORT_DUPLICATE_COLUMN = "METADATA_IMPORT_DUPLICATE_COLUMN"
    METADATA_IMPORT_SCHEMA_UNSUPPORTED = "METADATA_IMPORT_SCHEMA_UNSUPPORTED"
    SOURCE_LOCATION_UNAVAILABLE = "SOURCE_LOCATION_UNAVAILABLE"
    NOT_SUPPORTED_IN_P0 = "NOT_SUPPORTED_IN_P0"


class ImportMode(str, Enum):
    preview = "preview"
    commit = "commit"


class ImportChangeType(str, Enum):
    added = "added"
    updated = "updated"
    unchanged = "unchanged"
    stale_candidate = "stale_candidate"
    conflict = "conflict"


class ImportStatus(str, Enum):
    preview_ready = "preview_ready"
    committed = "committed"
    failed = "failed"


class MetadataObjectRef(StrictBaseModel):
    catalog: str = "default"
    schema: str = "default"
    table: str
    column: str | None = None

    @property
    def table_fqn(self) -> str:
        return f"{self.catalog}.{self.schema}.{self.table}"

    @property
    def column_fqn(self) -> str | None:
        if self.column is None:
            return None
        return f"{self.catalog}.{self.schema}.{self.table}.{self.column}"
```

### 3.2 SourceLocation 与诊断模型

```python
class SourceLocation(StrictBaseModel):
    location_id: str
    entity_id: str
    entity_type: EntityType
    start_line: int | None = Field(default=None, ge=1)
    start_col: int | None = Field(default=None, ge=1)
    end_line: int | None = Field(default=None, ge=1)
    end_col: int | None = Field(default=None, ge=1)
    start_offset: int | None = Field(default=None, ge=0)
    end_offset: int | None = Field(default=None, ge=0)
    raw_text: str | None = None
    confidence: float = Field(default=0.0, ge=0.0, le=1.0)


class Diagnostic(StrictBaseModel):
    diagnostic_id: str
    code: DiagnosticCode
    level: DiagnosticLevel
    message: str
    suggestion: str | None = None
    source_location_id: str | None = None
    related_entity_ids: list[str] = Field(default_factory=list)
    details: dict[str, Any] = Field(default_factory=dict)


class DiagnosticsReport(StrictBaseModel):
    diagnostics: list[Diagnostic] = Field(default_factory=list)
    error_count: int = 0
    warning_count: int = 0
    info_count: int = 0

    @field_validator("error_count", "warning_count", "info_count")
    @classmethod
    def counts_must_not_be_negative(cls, value: int) -> int:
        if value < 0:
            raise ValueError("diagnostic counts must not be negative")
        return value
```

## 5. P0 API 规格

### 5.1 通用规则

| 项 | 规格 |
|---|---|
| Base URL | `/api` |
| 请求格式 | `application/json`，GET 除外 |
| 响应格式 | `application/json` |
| 鉴权 | P0 不做 |
| CORS | 本地开发允许前端 dev server origin |
| 请求校验失败 | HTTP 422 |
| 未捕获异常 | HTTP 500 + `ApiErrorResponse` |
| 分析类业务失败 | HTTP 200 + `AnalysisResult.status = failed` |

统一错误响应：

```json
{
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "Unexpected server error",
    "details": {}
  }
}
```

### 5.2 `GET /api/health`

用途：前端启动时检查后端可用。

响应：

```json
{
  "status": "ok",
  "service": "sql-lineage-workbench",
  "version": "0.1.0"
}
```

验收：

```text
1. HTTP 200。
2. status 必须为 ok。
```

### 5.3 `POST /api/sql/format`

用途：格式化 SQL。P0 只要求简单 select 可格式化，格式化失败返回诊断。

请求：

```json
{
  "sql": "select order_no,user_id from default.order_table",
  "dialect": "spark"
}
```

成功响应：

```json
{
  "status": "success",
  "dialect": "spark",
  "formatted_sql": "SELECT\n  order_no,\n  user_id\nFROM default.order_table",
  "diagnostics": []
}
```

失败响应：

```json
{
  "status": "failed",
  "dialect": "spark",
  "formatted_sql": null,
  "diagnostics": [
    {
      "diagnostic_id": "diag:PARSE_ERROR:1",
      "code": "PARSE_ERROR",
      "level": "error",
      "message": "SQL 解析失败",
      "suggestion": "请检查 SQL 语法",
      "source_location_id": null,
      "related_entity_ids": [],
      "details": {}
    }
  ]
}
```

### 5.4 `POST /api/sql/analyze`

用途：P0 核心分析接口。

请求：

```json
{
  "sql": "select order_no, user_id from default.order_table",
  "dialect": "spark",
  "analysis_level": "column",
  "default_catalog": "default",
  "default_schema": "default",
  "metadata_version": "latest",
  "case_sensitive": false,
  "analysis_options": {
    "include_graph": true,
    "include_semantics": false,
    "include_diagnostics": true,
    "include_source_location": true,
    "include_expression_lineage": false
  }
}
```

成功响应必须包含：

```text
analysis_id
status
dialect
normalized_sql
metadata_context
lineage_ir
semantics_report
diagnostics_report
graph_view_model
source_locations
summary
```

P0 响应约束：

| 字段 | 约束 |
|---|---|
| `analysis_id` | 非空字符串，建议 UUID |
| `status` | `success` / `partial` / `failed` |
| `lineage_ir.nodes` | 至少包含表节点、源字段节点、输出字段节点 |
| `lineage_ir.edges` | 至少包含 projection / alias / expression |
| `semantics_report.status` | 默认 `not_supported_in_p0` |
| `diagnostics_report` | 永远返回 |
| `graph_view_model.supported_view_modes` | 永远包含五种 view mode |
| `source_locations` | P0 可为空，但字段必须返回 |

### 7.3 页面状态机

| 状态 | 触发 | UI 行为 |
|---|---|---|
| `idle_empty` | 首次打开页面 | 编辑器显示示例 SQL，画布显示空状态，诊断为空 |
| `metadata_missing` | 未导入任何元数据时点击分析 | analyze 可执行，但未知表会显示诊断；元数据面板提示导入 |
| `formatting` | 点击格式化 | 格式化按钮 loading，编辑器只读可选 |
| `analyzing` | 点击分析 | Analyze 按钮 loading，画布显示分析中，保留上一次成功图谱但弱化 |
| `analyze_success` | status = success | 更新画布、诊断面板、RawResult |
| `analyze_partial` | status = partial | 更新可用图谱，诊断面板自动打开并突出 error/warning |
| `analyze_failed` | status = failed | 画布显示失败空态，诊断面板自动打开 |
| `import_editing` | 打开导入弹窗 | JSON 编辑区可输入，preview 按钮可用 |
| `import_previewing` | 点击预览 | preview loading，不写库 |
| `import_preview_ready` | 预览成功 | 展示 changes 和 diagnostics，commit 按钮可用 |
| `import_committing` | 点击确认导入 | commit loading，禁止关闭弹窗或二次提交 |
| `import_committed` | 导入成功 | 展示摘要，刷新元数据版本，提示可重新分析 |
| `import_failed` | 导入失败 | 展示 diagnostics，不更新元数据版本 |

### 7.4 组件交互要求

#### HeaderToolbar

| 控件 | P0 行为 |
|---|---|
| 方言选择 | 默认 `spark`，可切换 `spark` / `hive` |
| 默认 schema | 默认 `default` |
| 格式化 | 调用 `/api/sql/format` |
| 分析 | 调用 `/api/sql/analyze` |
| 导入元数据 | 打开 MetadataImportDialog |

禁用规则：

```text
1. SQL 为空时禁用格式化和分析。
2. formatting / analyzing 时禁用格式化和分析按钮。
3. import_committing 时禁用导入弹窗关闭。
```

#### SqlEditorPanel

P0 能力：

```text
1. Monaco Editor 基础输入。
2. 支持 spark/hive 作为 language label。
3. 格式化成功后替换编辑器内容。
4. 分析失败后，如果 diagnostic 有 source_location_id 且能找到位置，则标注；否则只在诊断面板展示。
```

P0 不做：

```text
completion
hover
code action
精准 SQL 与图谱双向定位
```

#### LineageCanvasPanel

P0 使用 React Flow 展示 `graph_view_model.nodes` 和 `graph_view_model.edges`。

行为：

```text
1. 无分析结果：显示空状态。
2. analyzing：保留旧图，增加 loading 遮罩。
3. failed：清空图或保留旧图但明确标识失败，推荐清空当前失败结果图。
4. success / partial：渲染最新图。
5. 点击节点：选中节点，打开右侧详情抽屉或在底部元数据面板展示。
6. 节点拖拽：只影响前端位置，不回写后端。
7. fit view：每次新分析成功后自动执行一次。
```

节点样式：

| node_type | P0 样式要求 |
|---|---|
| `table` | 表名、schema，作为分组上游 |
| `column` | 字段名、数据类型、注释摘要 |
| `output_column` | 输出字段，突出显示 |
| `expression` | 表达式占位节点，显示表达式文本 |
| `unknown` | 警告样式 |
| `diagnostic` | P0 可不渲染 |

边样式：

| edge_type | P0 样式要求 |
|---|---|
| `projection` | 实线 |
| `alias` | 实线，label 显示 alias |
| `expression` | 实线或虚线，突出表达式加工 |
| `unknown` | 警告色虚线 |

#### DiagnosticsPanel

行为：

```text
1. 默认按 error、warning、info 排序。
2. partial / failed 时自动切换到底部 diagnostics tab。
3. 点击诊断：如果 source_location_id 可解析，编辑器定位；否则展示“P0 暂无精确位置”。
4. 每条诊断展示 code、level、message、suggestion。
```

空态：

```text
当前没有诊断问题
```

#### MetadataImportDialog

行为：

```text
1. 用户粘贴 JSON 后，前端先做 JSON.parse。
2. JSON.parse 失败时不请求后端，展示本地错误。
3. 预览成功后展示 changes 表格。
4. diagnostics 有 error 时禁用 commit。
5. commit 成功后刷新 metadata_version，并提示用户重新分析当前 SQL。
```

changes 表格列：

```text
change_type
object_type
catalog
schema
table
column
message
```

#### RawResultPanel

P0 必须提供，用于调试：

```text
1. 展示完整 AnalysisResult JSON。
2. 支持复制 JSON。
3. 不作为主交互入口。
```
