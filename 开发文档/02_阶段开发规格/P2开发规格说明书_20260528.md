# SQL 血缘解析工作台｜P2 开发规格说明书

生成日期：2026-05-28

## 1. 文档定位

本文档覆盖 P2 阶段需求，可直接交给 agent 在 P0/P1 工程基础上实现。

P2 范围：

```text
R13a ExpressionAnalyzer 基础抽取
R12 SemanticsReport 查询口径分析
R13b 表达式级血缘增强
R14 多视图图谱
```

P2 的目标是把系统从“字段血缘工具”升级为“SQL 理解工作台”。

## 2. P2 实现边界

P2 必须支持：

```text
1. 聚合表达式：sum / count / avg / min / max / count distinct
2. case when 表达式：区分条件依赖和取值依赖
3. 普通函数表达式：抽取参数字段，UDF 标记黑盒
4. 窗口函数：抽取函数主体、partition by、order by 字段
5. where / having 过滤条件
6. group by / distinct / window partition 粒度
7. join type / join keys 基础口径
8. 表级、字段级、表达式级、口径级、诊断级多视图
```

P2 暂不要求：

```text
1. 完整 SQL 优化器能力。
2. UDF 内部语义解析。
3. 企业指标平台对齐。
4. 复杂递归 CTE。
5. 跨 SQL 文件血缘。
6. 查询执行结果验证。
```

### 4.1 ExpressionAnalyzer

```

class ExpressionAnalyzer:
    def analyze(
        self,
        ast: Any,
        lineage_ir: LineageIR,
        metadata_context: MetadataContext,
        source_locations: list[SourceLocation],
    ) -> ExpressionReport:
        ...
```

解析规则：

| 表达式 | kind | 要求 |
|---|---|---|
| `col` | column | source_columns 包含 col |
| `col * 0.1` | binary | source_columns 包含 col |
| `sum(amount)` | aggregate | aggregation=sum，source_columns=amount |
| `count(distinct user_id)` | aggregate | aggregation=count_distinct，is_distinct=true |
| `case when status=1 then amount end` | case_when | condition_columns=status，value_columns=amount |
| `row_number() over(partition by user_id order by dt)` | window | window_spec 包含 partition/order |
| `my_udf(col)` | udf | source_columns=col，生成 UDF_BLACK_BOX warning |

ID 规则：

```text
expr:{scope_id}.{output_name}
expr:{scope_id}.where:{index}
expr:{scope_id}.having:{index}
expr:{scope_id}.window:{output_name}
```

分析规则：

| 能力 | 规则 |
|---|---|
| query_type | P2 支持 select / insert_select / ctas 的基础识别 |
| result_grain | 有 group by 为 group_by；有 distinct 为 distinct；否则 detail；无法判断为 unknown |
| filters | where / having 拆成 FilterItem |
| partition filter | 字段 is_partition=true 且出现在 where 等值或范围条件中 |
| metrics | 聚合表达式输出为 MetricItem |
| joins | 复用 P1 JoinReference |
| windows | 窗口表达式输出为 WindowItem |
| dedup_logic | distinct、row_number 过滤、group by 识别 |
| risks | 无分区过滤、未知粒度、UDF 黑盒、join 基数未知 |

请求新增 options：

```json

  "analysis_options": {
    "include_graph": true,
    "include_semantics": true,
    "include_diagnostics": true,
    "include_source_location": true,
    "include_expression_lineage": true,
    "include_all_graph_views": true
  }
}
```

响应新增：

```json
{
  "expression_report": {
    "status": "success",
    "expressions": []
  },
  "graph_views": {
    "default_view_mode": "column",
    "views": {
      "table": {},
      "column": {},
      "expression": {},
      "semantics": {},
      "diagnostics": {}
    }
  }
}
```

规则：

```text
1. include_semantics=false 时 semantics_report 可返回 not_supported_in_p0 或 partial 空结构。
2. include_expression_lineage=false 时 expression_report 可为 null。
3. include_all_graph_views=false 时只返回 graph_view_model。
```

### 5.2 `POST /api/graph/view`

用途：前端按需构建单个视图，避免 analyze 响应过大。

请求：

```json
{
  "analysis_result": {},
  "view_mode": "expression",
  "filters": {
    "visible_node_types": ["column", "expression", "output_column"],
    "visible_edge_types": ["expression", "aggregation"]
  }
}
```

响应：`GraphViewModel`

P2 最小实现：

```text
1. 可以只支持传 analysis_result。
2. 不要求从数据库加载历史分析。
3. GraphViewModel 坐标可由前端布局或后端简单分层布局生成。
```
