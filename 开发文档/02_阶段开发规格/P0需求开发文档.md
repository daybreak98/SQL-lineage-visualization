# P0 需求开发文档

生成日期：2026-05-28

## 1. 项目概述

基于P0开发规格说明书，实现SQL血缘解析工作台的后端和前端核心功能。

### 1.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 后端框架 | FastAPI + Uvicorn |
| SQL解析 | SQLGlot |
| 数据存储 | SQLite |
| 数据模型 | Pydantic v2 |
| 前端框架 | React 18 + TypeScript |
| 状态管理 | Zustand |
| 构建工具 | Vite |

### 1.2 P0实现范围

| 能力 | 状态 | 说明 |
|------|------|------|
| JSON元数据导入 | ✅ | preview/commit两种模式 |
| SQLite元数据查询 | ✅ | 表/字段搜索 |
| Monaco SQL输入 | ✅ | 基础textarea（Monaco待集成） |
| SQL格式化 | ✅ | SQLGlot transpile |
| SQL分析 | ✅ | 表/字段级血缘 |
| 血缘图展示 | ✅ | SVG基础画布 |
| 诊断报告 | ✅ | error/warning/info |

### 2.1 R001: Pydantic合同模型

**需求**: 实现P0规格说明书中的所有Pydantic数据模型

**实现步骤**:

1. 创建 `backend/app/domain/contracts.py`
2. 实现基础模型类 `StrictBaseModel`（禁止额外字段）
3. 实现枚举类型：
   - `AnalysisStatus`: success/partial/failed
   - `Dialect`: spark/hive/mysql/starrocks/doris
   - `EntityType`: 13种实体类型
   - `LineageNodeType`: 7种节点类型
   - `LineageEdgeType`: 5种边类型
   - `DiagnosticCode`: 14种诊断码
   - `ImportMode`: preview/commit
   - `ImportStatus`: preview_ready/committed/failed
4. 实现核心数据模型：
   - `MetadataObjectRef`: 元数据对象引用（含table_fqn/column_fqn属性）
   - `SourceLocation`: 源码位置
   - `Diagnostic`: 诊断信息
   - `DiagnosticsReport`: 诊断报告（含计数验证器）
   - `MetadataColumnInput/TableInput`: 元数据输入
   - `MetadataImportPayload/Request/Result`: 导入流程
   - `ResolvedTable/MissingTable/MissingColumn/AmbiguousColumn`: 解析结果
   - `MetadataContext`: 元数据上下文
   - `AnalyzeSqlRequest`: 分析请求
   - `ScopeItem`: 作用域
   - `LineageNode/Edge/IR`: 血缘数据
   - `GraphNode/Edge/ViewModel`: 图数据
   - `AnalysisResult`: 完整分析结果
5. 实现辅助API模型：
   - `FormatSqlRequest/Response`
   - `MetadataTableSearchResponse/ColumnSearchResponse`
   - `ApiError/ApiErrorResponse`

**测试验证**:

```python
# 序列化/反序列化测试
from app.domain.contracts import AnalysisResult, AnalyzeSqlRequest

# 创建请求
request = AnalyzeSqlRequest(sql="SELECT 1", dialect="spark")
assert request.sql == "SELECT 1"
assert request.dialect.value == "spark"

# JSON序列化
json_data = request.model_dump()
assert "sql" in json_data

# 反序列化
request2 = AnalyzeSqlRequest(**json_data)
assert request2.sql == request.sql
```

```python
from app.services.sql_parse_service import SqlParseService
from app.domain.contracts import Dialect

parser = SqlParseService()

# 解析成功
ast, diags = parser.parse("SELECT order_no FROM default.order_table", Dialect.spark)
assert ast is not None
assert len(diags) == 0

# 格式化
from app.domain.contracts import FormatSqlRequest
result = parser.format(FormatSqlRequest(sql="select 1", dialect=Dialect.spark))

assert "SELECT" in result.formatted_sql

# 解析失败
ast, diags = parser.parse("SELECT FROM", Dialect.spark)
assert ast is None
assert len(diags) > 0
```

### 2.4 R004: 血缘分析引擎

**需求**: 实现SQL血缘分析核心逻辑

**实现步骤**:

1. 创建 `backend/app/services/lineage_engine.py`
2. 实现 `analyze()` 主方法：
   - SQL解析
   - 元数据加载
   - 表解析
   - 字段血缘构建
   - 图构建
   - 状态判定
3. 实现 `_resolve_tables()`：
   - 匹配元数据中的表
   - 生成MissingTable和诊断
4. 实现 `_build_column_lineage()`：
   - 解析SELECT字段
   - 匹配源表字段
   - 处理别名
   - 处理未知字段
   - 处理歧义字段
5. 实现 `_build_graph()`：
   - 节点布局（表→源字段→输出字段）
   - 边构建
6. 实现 `_build_diagnostics_report()`

**测试验证**:

```python
from app.db.metadata_db import MetadataDatabase
from app.services.lineage_engine import LineageEngine
from app.domain.contracts import AnalyzeSqlRequest, ImportMode, MetadataImportPayload

# 初始化
db = MetadataDatabase(":memory:")
engine = LineageEngine(db)

# 导入元数据
payload = MetadataImportPayload(
    schema_version="1.0",
    metadata_version="test-v1",
    tables=[{
        "catalog": "default",
        "schema": "default",
        "name": "order_table",
        "columns": [
            {"name": "order_no", "data_type": "string"},
            {"name": "user_id", "data_type": "string"},
        ]
    }]
)
db.import_metadata(ImportMode.commit, payload)

# 分析
request = AnalyzeSqlRequest(
    sql="SELECT order_no, user_id FROM default.order_table",
    metadata_version="test-v1"
)
result = engine.analyze(request)

assert result.status.value == "success"
assert result.summary.table_count == 1
assert result.summary.output_column_count == 2
```

## 7. 已知限制

### 7.1 P0不做（按规格）

- 复杂CTE血缘
- 深层子查询
- 完整SELECT *展开
- completion/hover
- 完整表达式级血缘
- 完整查询口径分析
- 多视图切换实现
- SQL diff
- 历史快照
- 真实SQL执行

### 7.2 当前实现简化

- SQL编辑器使用textarea（Monaco待集成）
- 图布局使用固定坐标（自动布局算法待优化）
- 表达式血缘仅支持简单字段引用
- 未实现SourceLocation精确定位
