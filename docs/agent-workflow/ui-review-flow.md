# UI 审阅流转规则

## 概述

本文档定义了 SQL 血缘解析可视化项目的 UI 审阅流转规则，确保 UI 优化建议的合理采纳。

---

## 核心规则

### 1. UI Agent 不直接写代码

- senior-ui-designer 只输出 UI/UX 审阅报告
- 不直接修改前端实现文件
- 不直接修改产品功能定义

### 2. 每次前端功能完成后，UI Agent 才进行审阅

- 触发条件：senior-frontend-developer 完成功能实现
- 由 project-orchestrator 调度 senior-ui-designer 进行审阅

### 3. UI Agent 输出美化变更文档

输出内容包括：
- 当前界面问题
- 用户注意力问题
- 信息密度问题
- 视觉层级问题
- 建议优化项
- 优先级（P0/P1/P2）

### 4. 产品经理判断是否采纳

- senior-product-manager 收到 UI 审阅报告
- 产品经理根据以下标准判断：
  - 是否影响核心功能
  - 是否符合产品方向
  - 是否值得投入资源
  - 优先级是否合理

### 5. 采纳项写入下一轮前端迭代文档

- 产品经理将采纳的优化项写入迭代文档
- 明确标注为 UI 优化任务
- 分配优先级和迭代周期

### 6. 下一轮前端开发时由前端开发实现

- senior-frontend-developer 在下一轮迭代中实现
- 按照迭代文档中的 UI 优化任务执行

### 7. 未采纳项归档，不直接进入开发

- 未采纳的优化项归档保存
- 可在后续迭代中重新评估
- 不直接进入开发流程

---

## UI 审阅流程图

```
senior-frontend-developer 完成前端功能
  ↓
project-orchestrator 调度 UI 审阅
  ↓
senior-ui-designer 输出 UI/UX 审阅报告
  ↓
senior-product-manager 收到报告
  ↓
产品经理判断是否采纳
  ├─ 采纳 → 写入下一轮前端迭代文档
  └─ 不采纳 → 归档保存
  ↓
下一轮迭代
  ↓
senior-frontend-developer 实现 UI 优化
```

## SQL 血缘可视化项目 UI 审阅重点

### SQL 编辑器体验

- 语法高亮清晰度
- 自动补全提示
- 错误提示位置和样式

### 血缘画布体验

- 节点布局清晰度
- 连线样式和可读性
- 拖拽交互流畅度
- 缩放和平移操作

### 元数据展示

- 表结构信息层级
- 字段注释可读性
- 血缘关系展示

### 整体布局

- 编辑器和画布的空间分配
- 侧边栏信息组织
- 响应式适配
