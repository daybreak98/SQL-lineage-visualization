# SQL 血缘解析可视化项目资料索引

本目录当前是项目资料与需求文档工作区，按“目标、开发文档、测试用例”三类组织。

## 目录结构

```text
.
├── 项目目标/
│   └── 最终目标.md
├── 开发文档/
│   ├── 00_主需求文档/
│   ├── 01_Agent模块拆解文档/
│   ├── 02_阶段开发规格/
│   ├── 03_前端页面文档/
│   └── 99_历史文档/
└── 测试用例/
    ├── 00_血缘解析基准/
    ├── 99_非业务元数据/
    └── 各业务 SQL 样例目录
```

## 推荐阅读顺序

1. `项目目标/最终目标.md`
2. `开发文档/00_主需求文档/sql_lineage_workbench_requirement_breakdown_v0.9.1.md`
3. `开发文档/01_Agent模块拆解文档/README.md`
4. `开发文档/01_Agent模块拆解文档/00_AGENT_MODULE_INDEX.md`
5. 按模块编号阅读 `开发文档/01_Agent模块拆解文档/*.md`

## Agent 实现入口

给 agent 开发时，优先使用：

```text
开发文档/01_Agent模块拆解文档/
```

其中 `00_AGENT_MODULE_INDEX.md` 是执行入口，列出了 M01-M28 的依赖顺序和回归要求。

`开发文档/02_阶段开发规格/` 是阶段级补充规格，适合在模块文档不够细时参考。

## 维护规则

- 当前主线文档放在 `00_主需求文档`、`01_Agent模块拆解文档`、`02_阶段开发规格`。
- 旧版本、评审报告、历史评价放在 `99_历史文档`。
- 测试 SQL 和业务样例放在 `测试用例`，IDE 元数据放在 `测试用例/99_非业务元数据`。
- 不要把旧版技术方案放回主线目录，避免 agent 误读。
